## What happens
The release request arrives in the shared Operations queue in the payments workflow tool. Each queue item carries a facility ID, amount, currency, value date, beneficiary details and the supporting documents — the drawdown notice and invoice. An item enters the queue by one of three paths: a front-office drawdown request against an approved facility; the workflow tool auto-generating the item when a held payment reaches its scheduled release date; or an Operations Analyst manually creating the item for a flagged release.

## Inputs
- Front-office drawdown request against an approved facility, with supporting documents attached
- A held payment reaching its scheduled release date — item auto-generated by the payments workflow tool with no human actor
- A flagged release manually initiated by an Operations Analyst — flagged by the daily held-vs-released reconciliation (CP-FR-005) or by the lifting of a Compliance hold
- Supporting documents carried with the request: the drawdown notice and the invoice

## Outputs
- Release request item in the shared Operations queue
- Facility ID, amount, currency, value date and beneficiary details captured on the item
- Supporting documents — the drawdown notice and invoice — attached to the item

## Why it matters
Capturing every release request — with its full data set and its supporting documents — in a single queue from a known starting point ensures each item can be tracked, routed and controlled regardless of which of the three entry paths created it.
