## Context
SCT Inst, SCT and SDD all move to ISO 20022 by November 2025. The existing internal format is a proprietary fixed-width record that has accreted dozens of field aliases. Multiple translations between formats add latency and rule duplication.

## Decision
Adopt ISO 20022 pain.001 / pacs.008 as the canonical internal message between every component from channel ingress through the CSM Gateway. Translate only at the edges.

## Alternatives considered
- Keep the proprietary internal format and translate at the CSM boundary only.
- Adopt ISO 20022 for inter-bank messages only, retain proprietary internally.
- Wait for the 2027 SWIFT MX cutover and migrate everything at once.
- Use a generic event envelope with payment payload nested inside.

## Consequences
- One canonical model removes a class of translation bugs.
- Schema validation can sit at every component boundary.
- Vendor extensions still need a bilaterally-agreed namespace.
- Larger messages on the wire — bandwidth modelling matters at peak.
- Channels that emit proprietary today need an edge translator built first.
