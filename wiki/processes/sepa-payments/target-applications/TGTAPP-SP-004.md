## Rationale
Stick with the current CSM Gateway product but upgrade it to ISO 20022-native and add SCT Inst endpoints. A rip-and-replace would require fresh scheme certification, which is not a price worth paying.

## Tech stack
SWIFT Alliance Gateway with the bank's existing CSM Gateway product, ISO 20022 message library, IBM MQ for the legacy CSM endpoints during transition.

## Risks
- SWIFT scheme certification windows are narrow; missing one slips by a quarter.
- Test environment availability with the CSM is the binding constraint.
- Vendor's ISO 20022 release calendar is on the critical path.
