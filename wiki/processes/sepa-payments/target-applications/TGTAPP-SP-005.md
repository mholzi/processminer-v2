## Rationale
No existing component pushes transactional payment notifications. Build a thin event-driven service shared across products instead of letting each channel team poll the core for outcomes.

## Tech stack
Node.js + Kafka consumers on Kubernetes, Firebase Cloud Messaging and Apple Push Notification Service for mobile, Twilio for SMS fallback and Redis for delivery state.

## Risks
- Push deliverability outside the EU is variable — fallbacks must be reliable.
- Mobile app teams' release cadence sets the integration timeline.
- Apple and Google push token churn requires a robust re-registration path.
