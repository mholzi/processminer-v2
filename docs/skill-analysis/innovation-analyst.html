innovation-analyst

Refines the web-sourced forward-looking layer of a process with the SME — market trends, what competitor banks and fintechs are doing, innovation ideas — and weighs the risk of pursuing each, grounding every idea in a real documented pain or friction.

Interactive · Y/E/R loop LENS: forward-looking — trends, competitor moves, ideas, risk OWNS: market-trend · competitor-innovation · innovation-idea · innovation-risk Invoked by: free chat / Deep Dive / qer-session

a What it does

innovation-analyst owns the Innovation perspective of a process. It is a refinement specialist far more than a generator: market trends, the three competitor scans (EU / global / fintech) and innovation ideas are usually web-sourced first by the non-interactive source-innovation skill, and this skill starts from those drafts — walking each with the SME, correcting it, and adding what the sourcing missed. The one element type it builds from scratch with the SME is the innovation-risk: the honest downside of pursuing an idea.

It is disciplined, not free-associating. Every innovation-idea must addresses a real documented pain-point or friction-point — an idea that solves no recorded problem is a prompt to find the problem, not a finding. Ideas trace back to the market-trend or competitor move they derive from (fromTrend), so the innovation view is a graph that later feeds the transformation-agent's Target Process, not a wish list.

Key constraints and behaviours:

  • No web research itself. Sourcing is source-innovation's job. When the SME wants a new market-trend or competitor move the skill cannot cite, it still writes the element — but with sourceUrl: pending (the literal word), status: draft and confidence: low, and lists every such element in the close-out for a later sourcing pass.
  • Preserve sourced frontmatter on edit. On [E], patch only the corrected block via updateElement — never re-author the element — so a trend's sourceUrl, asOf, horizon, bearsOn and an idea's fromTrend survive untouched.
  • Honest risk. An upside with no downside named is incomplete; every idea earns at least one innovation-risk (adoption, regulatory, delivery, dependency).
  • Stays in its lane. It reads the As-Is freely as input but never authors As-Is, CX, control or systems elements; the Target Process belongs to the transformation-agent, which it hands off to.
  • Everything is status: draft with an honest confidence; the SME triages relevance and approves in the app.

b The sequence

Five phases run in order. Under orchestrated mode (called by qer-session) Phase 0 and Phase 4 are skipped, but Phase 1 always runs — the skill always reads the As-Is and the sourced trends first.

  1. Phase 0 — SetupUse the SME name/role from the invocation; only ask if absent. Identify the process from the slugs and read its overview. Tools: expandElement()
  2. Phase 1 — OrientationRead the documented pain-points and friction-points and the existing web-sourced market-trend / innovation-idea elements; confirm with the SME which pains hurt most. If nothing is sourced yet, point them at source-innovation first. Tools: expandElement()
  3. Phase 2 — Refine trends, competitors & ideasWalk each existing trend / competitor move / idea one at a time, running Y/E/R; on [E] patch only the corrected field to keep sourced frontmatter intact. Then ask what is missing and draft with the [A]/[E]/[N] idiom; uncitable new trends get sourceUrl: pending. Tools: updateElement() createElement()
  4. Phase 3 — Innovation risksFor each idea, build innovation-risk elements from scratch with the SME via [A]/[E]/[N] — adoption, regulatory, delivery, dependency. An idea with no risk named is incomplete. Tools: createElement()
  5. Phase 4 — Validation & close-outSweep for ideas addressing no problem, ideas with no risk, trends nothing links to; surface each as a short question, then emit the canonical close-out with per-type counts and a separate list of every sourceUrl: pending element. Point the SME at transformation-agent next. Tools: checkConformance()

Biggest win: turn Phase 2 from a serial Y/E/R walk into a deterministic batch triage over the source-innovation drafts. The skill's real job is judgement on top of already-cited material; reviewing a screen of trends and competitor moves in one keep/drop/edit pass — with read-back skipped where provenance=web already carries evidence — removes the bulk of the round-trips while keeping every human decision intact. Pair it with a precomputed pain/friction index so addresses links are wired by id, and the skill becomes both far faster and far more reproducible.
