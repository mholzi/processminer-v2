→ The spine

The backbone flow for documenting a process, and where the rest hang off it:

new-process → document-ingest → [triage] → foundational-run → perspective specialists → transformation-agent → council-review

qer-session is the authored conductor over the whole thing; run-lint is the consistency checkpoint you can run any time; the source-* skills pre-fill perspectives from the web; the dtp-* skills work the procedure document; dogfood-run drives all of it as a QA harness.

∑ Cross-cutting enhancement themes

The same determinism-and-speed levers recur across the roster. The per-skill pages give the specifics; these are the patterns worth funding once and applying everywhere.

Parallel sub-agent fan-out — run independent units of work concurrently instead of seriallydet + speedrun-lint & council-review (the five lenses are already conceptually parallel), document-ingest, area-summary, the source-* skills

The single highest-leverage move across the roster is literal parallel sub-agent fan-out backed by strict output schemas: the lint council, the target-state council, document ingest and the web-sourcing skills are all written as "do N independent things," yet run serially. Fanning them out collapses wall-clock to the slowest unit, and the schema makes every unit's output deterministic and auditable — determinism and speed from one change.
