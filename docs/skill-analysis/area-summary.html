area-summary

Reads every section in one area of a process and writes an Amazon-style narrative memo — four fixed headings, full prose, grounded in real ids and figures — into the process's summaries field for the app to render. Non-interactive: no SME questions, no approval loop.

Non-interactive (read, write, report) WRITES_TO: summaries field via writeSummary() Invoked by: a button SCOPE: one area

a What it does

area-summary generates an executive summary of one area of a process (<slug> + <area>, one of as-is, risk-compliance, client-experience, innovation, target, it-architecture). It reads every element across all of the area's sections plus the process overview, then writes a memo and stores it where the app's summary panel renders it. It is a silent generation, like source-cx — no SME questions, no Y/E/R, no approval.

The memo style is non-optional: an Amazon-style narrative — full sentences and paragraphs, no bullets, that tells a story (context → substance → honest read → recommendation), specific and measurable ("three of the eight steps carry no control" beats "some steps lack controls"), and standing on its own so a reader understands the whole area in a couple of minutes. It uses exactly four ## headings in order: Introduction, Current state, What stands out, Recommendation.

Every statement must trace to the area's actual elements — nothing invented. An area with no elements gets a summary that plainly says it is not yet documented, in the same narrative voice. The writeSummary tool validates that the memo has exactly those four headings in order and errors otherwise. The skill summarises one area per run and writes nothing else.

b The sequence

Read the whole area, write the narrative memo, store it — three steps, no loop.

  1. Read the areaFind the area in the schema and its sections, then read every element across all of them plus the process overview for domain context. An empty area is noted as not-yet-documented. Tools: expandElement({ type }) · expandElement({ type, id })
  2. Write it as an Amazon-style narrative memoCompose flowing prose under exactly four headings in order — Introduction, Current state (the heart and longest part), What stands out (the candid read), Recommendation. No bullets; name elements by title, cite ids and real figures; ground every statement in actual elements.
  3. Store itPass the markdown to the writer, which stores it in summaries keyed by area and checks the four headings are present and in order — if it errors, fix the headings and rerun. Then report exactly one line. Tools: writeSummary({ slug, area, summary })
