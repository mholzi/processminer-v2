Found while running `source-cx` on the Debit Card Replacement process.

The web-sourcing skills (`source-cx`, `source-innovation`, `source-regulation`)
tell the agent to call `show_template.py <type>` and treat its output as
authoritative — `source-cx` says verbatim:

> "That is the full contract — you do **not** read the whole schema file."

But for `competitor-cx-eu` (and `-global`, `-fintech`, `cx-benchmark`)
`show_template.py` prints:

    fields:
      competitor
      asOf
      source
    ...
    required:  competitor, asOf, sourceUrl

`sourceUrl` appears in `required` but **nowhere else** — not in `fields`, not
in the universal-keys header. A skill that obeys the instruction and reads
only `show_template` is told a field named `sourceUrl` is required, but is
given no label, no hint, no format, and no indication it is even a frontmatter
key. It cannot satisfy the contract from the contract.

**Why it happens.** In `schema/process-schema.json` the `source` field carries
a companion:

    { "key": "source", "label": "Source",
      "hint": "Citation or URL for the observation.",
      "urlKey": "sourceUrl" }

`source` is a *paired* field — a citation plus a URL stored under `sourceUrl` —
and `required` lists the URL half. `show_template.py` renders the `fields`
list but **drops the `urlKey` companion entirely**, so the pairing is invisible
in its output.

**How it bit this run.** Writing the five CX elements, the run had to *infer*
that `sourceUrl` is a frontmatter field and pass it inside the
`write_element.py` spec's `fields` map. It happened to work — `write_element`
accepts the key — but only because the schema file was read directly,
the one thing the skill says is unnecessary. An agent that trusted
`show_template` alone would write elements missing a required field and not
know why conformance later complained.

**Suggested fix.** Have `show_template.py` surface the `urlKey` companion — e.g.
render `source` as `source (+ sourceUrl: citation URL, required)` — so the
printed contract actually contains every required key. While there: the
`required` list names `sourceUrl` but not `source`, so an element with a bare
URL and no citation text conforms; consider requiring the pair together.
