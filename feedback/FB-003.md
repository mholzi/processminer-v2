Ingesting the Corporate Account Closure DTP ran as a single chat turn that
took roughly 25–30 minutes to extract 38 elements. Two problems with the
feedback the SME gets during that time:

1. **No progress count.** The live activity line only ever showed generic
   strings — "Searching files …", then "✏ Writing wiki element …". For the
   first ~10 minutes nothing was written to disk at all and the line sat on
   "Searching files …". There is no way to tell "extracted 12 of ~40" from
   "hung". This is the single longest operation in the app and it is the one
   with the least progress visibility.

2. **It looks indistinguishable from a stall.** I had to check the OS process
   list to confirm the headless run was still alive. An SME cannot do that.

Suggestion: have document-ingest emit a periodic progress line (elements
written so far / sections done), or have the route surface a coarse percentage
from the run manifest. Even "Extracting Process Steps (3/8 sections)" would be
a large improvement over a frozen "Searching files …".
