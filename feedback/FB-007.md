The non-interactive web-sourcing skills (source-regulation, source-cx,
source-innovation) run outside the chat — by design — and show a banner
"Sourcing from the web — this can take a couple of minutes" on the section
that launched them. That part works well.

The gap: the banner is scoped to that one section. These runs took several
minutes each in the F2B run (source-cx and source-innovation were the better
part of 10+ minutes each). If the SME navigates to another section or area
while it runs, there is no global indicator that a sourcing job is in
progress — and the chat shows nothing either, since web-sourcing deliberately
does not touch the transcript.

Suggestion: add a small global busy indicator (e.g. on the top bar, near the
section the run targets) so the SME can move around the wiki during a
multi-minute sourcing run without losing track of it, and gets notified where
to look when it finishes.

Minor, related: the re-ingest review message phrased the conflict scope as
"EUR 250,000 appears in six places across the wiki (PS-CAC-005, CP-CAC-004,
PG-CAC-001, TS-CAC-004, TD-CAC-005)" — six places but five element IDs
(CP-CAC-004 contributes two). "Six blocks across five elements" would read
more clearly.
