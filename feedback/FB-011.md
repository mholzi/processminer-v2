`document-ingest` Step 3 requires each block's provenance to carry an
`evidence` quote: a `document` heading must have "`evidence` set to the
verbatim supporting quote from the document". This `evidence` quote is the
backbone of the hallucination countermeasure — it is the thing a reviewer
trusts when a heading is marked `document` or `elicited`.

**Nothing checks it.** `write_element.py` writes whatever `evidence` string
the spec hands it. `check_conformance.py` validates blocks, headings and
frontmatter — it does not open the source document and confirm the quote is
actually in it. So an `evidence` value can be:

- a paraphrase rather than a verbatim quote,
- empty on a heading marked `document`,
- or a quote from the **wrong document** entirely.

This is not hypothetical. During this run's first ingest, the generator
script carried a leftover `evidence` quote from the *Direct Debit Mandate
Management* mockup ("carries the mandate reference ...") into a Debit Card
Replacement element. It was caught and corrected only by eye — `check_conformance`
passed it cleanly, because conformance never looks at the quote's content.

The whole provenance scheme (HALLUCINATION-PLAN.md) rests on `evidence` being
real. A guard that is never verified is a guard on trust alone.

**Suggested improvement.** Add a check — in `check_conformance.py` or a new
`check_provenance.py` run by the wiki test suite — that for every heading
marked `document`, the `evidence` substring actually occurs in the recorded
source file under `raw-sources/<slug>/`. A non-empty `evidence` on a
`proposed` heading, or a `document` heading with empty `evidence`, should also
flag. It cannot validate `elicited` quotes (the SME is not a file), but it can
make the `document` half of the countermeasure enforceable instead of
aspirational.
