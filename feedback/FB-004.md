Mid-way through the foundational run (after approving item 1, before item 2),
the chat panel reset to a single "Welcome back … paused at item 2 of 39"
resume banner. The entire conversation transcript — roughly a dozen messages,
including the SME's own typed answers — was gone.

Cause: the chat `messages` array and the `--resume` `chatSessionId` are both
ephemeral React state in `ProcessDocScreen.tsx`. Any remount drops them — a
genuine browser refresh, or in dev a hot-reload. `resumeMessage()` then
re-seeds the deterministic "welcome back" banner from `review-state.json`.

What survives: the foundational-run *state* (review-state.json on disk), so no
approved work was lost and the run resumed correctly. What does not survive:

- the conversation transcript (the SME loses the thread of what was discussed);
- the `--resume` session id, so the *next* chat turn starts a brand-new
  `claude` session with no memory of the prior conversation.

For a multi-hour elicitation tool this is fragile. Consider persisting the
transcript + session id (sessionStorage at minimum, or a per-process sidecar)
so a reload restores the conversation rather than silently discarding it.
