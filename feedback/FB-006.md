After running the transformation-agent to refine the Target Process stub, the
close-out message said:

> "Drafted: 15 element(s) … Everything is status: draft — review and approve
> the elements in the web app. Approval is your decision there."

…but the very same message then printed a table where every one of the 15
elements is marked "✓ M. Berger" ("firmed up"), and on disk all 15 target
elements were already `approval: approved` / `approvalBy: M. Berger`.

So the boilerplate close-out paragraph ("everything is draft, approve it
yourself") contradicts what the run actually did (it approved them, as the
SME had asked it to). The standard close-out template appears to assume the
non-interactive `source-target` stub path and is printed verbatim even when
transformation-agent has already approved elements with the SME. The summary
the SME reads should reflect the real end state.
